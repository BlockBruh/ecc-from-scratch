# Elliptic Curve Cryptography — from scratch in Rust

This repository is a personal learning project that demonstrates my understanding of the
mathematics behind **Elliptic Curve Cryptography (ECC)** and the **ECDSA** digital
signature scheme.  Everything is implemented from first principles using only `BigUint`
arithmetic — no external crypto libraries are used.

---

## What this demonstrates

- Arithmetic in a **prime finite field F_p**
- **Elliptic curve point arithmetic** (addition, doubling, scalar multiplication)
- The **double-and-add** algorithm for efficient scalar multiplication
- **ECDSA** key generation, signing, and signature verification
- Correctness verified against the real-world curves **secp256k1** (Bitcoin) and **secp256r1** (NIST P-256)

## Mathematical background

### Finite fields — `src/finite_fields.rs`

A finite field **F_p** is the set of integers `{0, 1, …, p−1}` where all operations are
performed modulo a prime `p`.

| Operation | Formula |
|---|---|
| Addition | `(a + b) mod p` |
| Subtraction | `(a − b) mod p = (a + (p − b)) mod p` |
| Multiplication | `(a · b) mod p` |
| Multiplicative inverse | `a⁻¹ ≡ a^(p−2) mod p` (Fermat's Little Theorem, requires `p` prime) |
| Division | `a / b = a · b⁻¹ mod p` |

### Elliptic curves — `src/ec.rs`

An elliptic curve over **F_p** is defined by the short Weierstrass equation:

```
y² ≡ x³ + a·x + b  (mod p)
```

Together with a special **point at infinity** (the group identity), the curve points form
an **abelian group** under the following addition law:

**Adding two distinct points P = (x₁, y₁) and Q = (x₂, y₂):**
```
slope  s = (y₂ − y₁) · (x₂ − x₁)⁻¹  mod p
x₃     = s² − x₁ − x₂                mod p
y₃     = s · (x₁ − x₃) − y₁          mod p
```

**Doubling a point P = (x₁, y₁) — tangent line formula:**
```
slope  s = (3·x₁² + a) · (2·y₁)⁻¹   mod p
x₃     = s² − 2·x₁                    mod p
y₃     = s · (x₁ − x₃) − y₁          mod p
```

**Scalar multiplication** `d · P` is computed with the **double-and-add** algorithm,
which processes the bits of `d` from most-significant to least-significant:
```
t = P
for each remaining bit b of d (high to low):
    t = double(t)
    if b == 1:  t = add(t, P)
return t
```
This runs in O(log d) point operations.

### ECDSA — `src/ecdsa.rs`

ECDSA (Elliptic Curve Digital Signature Algorithm) is built on top of the group
structure above.  Domain parameters are a curve, a generator point `G`, and the
order `n` of the subgroup generated by `G`.

**Key generation:**
```
private key  d  ← random scalar in [1, n)
public key   B  = d · G
```

**Signing** a message hash `z` with ephemeral nonce `k`:
```
R        = k · G
r        = R.x mod n
s        = (z + d · r) · k⁻¹  mod n
signature = (r, s)
```
> The nonce `k` must be random and **never reused** — reusing it leaks the private key.

**Verification** of `(r, s)` given public key `B` and hash `z`:
```
u1  = s⁻¹ · z  mod n
u2  = s⁻¹ · r  mod n
P   = u1 · G + u2 · B
valid  ⟺  P.x == r
```


Tests cover:
- Field arithmetic edge cases

## Limitations

This is an **educational implementation** — it is not production-safe:

- No constant-time guarantees (vulnerable to timing side-channels)
- `add` and `double` are separate operations; the caller must ensure the right one is used
- `inverse_mul_prime` uses Fermat's Little Theorem and requires a prime modulus
- No input validation beyond basic range checks
